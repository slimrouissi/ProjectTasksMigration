SECURITY CONSIDERATIONS
==============================================================================

SECRETS MANAGEMENT
  - appsettings.json contains secrets (in .gitignore)
  - Never commit secrets to version control
  - Store ClientSecret securely in production
  - Rotate secrets annually

AUTHENTICATION
  - OAuth2 client credentials flow
  - 60-minute JWT token with 5-minute refresh buffer
  - MSAL library for token management
  - Token stored in memory only

AUDIT TRAIL
  - All API calls logged to D365 audit logs
  - Application User tracked for all operations
  - Can review in D365 Settings → Audit

API PERMISSIONS
  - Requires Dynamics CRM API access
  - user_impersonation scope
  - Admin consent required

PERFORMANCE CHARACTERISTICS
==============================================================================

EXPORT PERFORMANCE
  - Small project (10 tasks):      ~2-3 seconds
  - Medium project (100 tasks):    ~5-10 seconds
  - Large project (1000+ tasks):   ~30-60 seconds
  - 190 projects total:            ~30-60 minutes

IMPORT PERFORMANCE
  - Depends on batch size and target environment
  - Small project (10 tasks):      ~3-5 seconds
  - Medium project (100 tasks):    ~10-20 seconds
  - Large project (1000+ tasks):   ~1-2 minutes
  - 190 projects total:            ~2-4 hours
  - Batch size 200 recommended

POLLING
  - Default interval: 5 seconds
  - Default max attempts: 120 (10 minutes total)
  - Typically completes in 1-2 polling attempts

MEMORY USAGE
  - Per project JSON: 100KB - 5MB
  - GUID mappings: ~1MB for 100K mappings
  - Total runtime: ~200-500MB for full migration

==============================================================================
TROUBLESHOOTING QUICK REFERENCE
==============================================================================

AUTHENTICATION ERRORS
  └─ Check: ClientId, ClientSecret, TenantId in config
  └─ Verify: Application User exists in D365
  └─ Confirm: API permissions granted in Azure AD

IMPORT FAILURES
  └─ Check: Custom field mappings correct
  └─ Verify: Lookup field values exist in target
  └─ Review: Logs for specific error messages

TIMEOUT ERRORS
  └─ Increase: PollingMaxAttemptsPerOperationSet
  └─ Decrease: BatchSize to 100 or 50
  └─ Check: Target environment performance

CONNECTIVITY
  └─ Test: dotnet run -- validate
  └─ Verify: Environment URLs are correct and accessible
  └─ Check: Network/firewall policies

GUID MAPPING
  └─ Resume uses saved mappings: guid_mappings.json
  └─ Mappings persist automatically after successful import
  └─ Reset with --resume flag (loads previous state)
