KEY FEATURES
================================================================================

ARCHITECTURE
• Canvas app is thin UI layer (NOT doing the migration)
• Power Automate flows are the migration engine
• Uses Schedule API (msdyn_OperationSet pattern) - ONLY supported method
• Custom Dataverse table tracks status for each project

CAPABILITIES
• Exports ALL project data from source (50+ custom fields)
• Includes tasks, team members, dependencies, assignments, buckets
• Handles pagination for >250 tasks per project
• Maps GUIDs for future reference data sync
• Creates projects in target using Schedule API (bulk operations)
• 2-phase import: Tasks first, then dependencies & assignments
• Maintains parent-child task relationships
• Supports 190+ projects

SCALABILITY
• Single project: 30 min - 2 hours
• 190 projects sequential: 10-15 days
• Avoids throttling with sequential processing
• Handles 500+ tasks per project
• Supports up to 50+ custom fields

ERROR HANDLING
• Try/Catch scopes on all critical sections
• Automatic retry for failed exports/imports
• Detailed error logging to migration tracker
• Resume capability (doesn't restart from beginning)
• Rollback instructions included

MONITORING
• Real-time dashboard in canvas app
• Status updates every 5 seconds
• Progress bars with task counts
• Error details visible on click
• Power Automate run history viewable
• Migration tracker queryable for reports

├─ Create cr_SourceDataverse connection (user auth)

└─ Create connection references
